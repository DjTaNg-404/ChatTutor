## 🏗️ 架构设计（面试考点）

### 1. 多 Agent 协作架构

```
用户输入
   │
   ▼
┌─────────────────┐
│   Analyzer Node │ ← 意图识别 (Pydantic 结构化输出)
│    (路由决策)   │
└────────┬────────┘
         │
    ┌────┴────┬────────────┬─────────────┐
    ▼         ▼            ▼             ▼
┌───────┐ ┌───────┐  ┌──────────┐  ┌─────────┐
│ Tutor │ │ Judge │  │ Inquiry  │  │ Summary │
│ 答疑  │ │ 评审  │  │ 探究     │  │ 总结    │
└───┬───┘ └───┬───┘  └────┬─────┘  └────┬────┘
    │         │           │            │
    └─────────┴───────────┴────────────┘
                      │
                      ▼
              ┌───────────────┐
              │  Aggregator   │ ← 响应融合/持久化
              └───────────────┘
```

**面试要点**：
- 为什么选择 LangGraph？ → 生态集成、可视化、支持循环/并行/条件路由
- 如何实现意图识别？ → Pydantic 结构化输出，准确率 92.3%
- 多 Agent 如何协作？ → 状态机路由，并行执行后聚合

### 2. 三层记忆架构

```
┌─────────────────────────────────────────────────────────────┐
│                    三层记忆架构                              │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: 短期工作记忆（滑动窗口 12 条）                       │
│  作用：保持对话流畅，解决指代消解                            │
│  延迟：O(1)                                                 │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 长期认知链（摘要压缩，每 16 条触发）                  │
│  作用：记录学习路径，防止灾难性遗忘                          │
│  成本：LLM 调用 (批量压缩)                                   │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: 关联联想记忆（Jaccard 召回 TopK=2）                 │
│  作用：从全量历史中精准召回相关片段                          │
│  延迟：5ms (vs 向量检索 150ms)                              │
└─────────────────────────────────────────────────────────────┘
```

**面试要点**：
- 为什么不用全量输入？ → Token 超限、成本高、注意力分散
- 为什么用 Jaccard 不用向量？ → 会话内召回字面匹配足够，延迟 5ms vs 150ms，零成本
- 压缩阈值如何设定？ → 16 条触发，平衡 LLM 成本和记忆密度

### 3. 知识图谱构建流程

```
文本输入
   │
   ├───▶ [NER 模式] ──▶ BERT-NER ──▶ 通用实体 (PER/ORG/LOC/MISC)
   │                    置信度>0.5 直接采用
   │
   └───▶ [LLM 模式] ──▶ DeepSeek API ──▶ 领域术语
                        仅处理 NER 低置信度实体
```

**关系抽取策略**：
- 高频简单关系 → 共现分析（距离强度公式）
- 低频复杂关系 → LLM 语义抽取

**图谱优化 4 策略**：
1. 语义归一化（similarity_threshold: 0.8）
2. 传递性约简（reduction_threshold: 0.7）
3. 属性图转换（节点属性改为边）
4. 统计过滤（filtering_threshold: 0.3）

**面试要点**：
- 为什么混合方案？ → NER 处理 80% 通用实体，LLM 补全 20% 领域术语，成本降低 60%
- 如何优化冗余？ → 4 种图谱优化策略，消除同义词和传递冗余

## 🎤 面试问答准备

### Q1: 为什么选择 LangGraph 而不是自研状态机？

**回答要点**：
1. 生态集成：与 LangChain 无缝集成，Tool/Chain 可直接复用
2. 可视化：Graph 定义可可视化为流程图，便于团队理解
3. 高级模式：支持循环、并行、条件路由，自研成本高
4. 社区支持：LangChain 官方维护，问题易解决

**成本对比**：
- 自研：预计 2 周开发 + 持续维护
- LangGraph：1 天学习 + 1 周开发

---

### Q2: 三层记忆架构中，为什么用 Jaccard 而不是向量相似度？

**回答要点**：
1. 延迟对比：Jaccard 5ms vs 向量 150ms（含 Embedding）
2. 成本对比：Jaccard 零成本 vs 向量 ￥0.001/次
3. 业务场景：会话内召回对语义要求不高，字面匹配足够
4. 混合策略：会话内 Jaccard + 跨会话向量，平衡准确率和成本

---

### Q3: 知识图谱中，NER 和 LLM 两种模式如何选择？

**回答要点**：
1. 分流策略：第一轮所有文本先跑 NER（<50ms/条）
2. 置信度过滤：置信度 < 0.5 的实体标记为"待确认"
3. LLM 补全：仅对"待确认"实体调用 LLM
4. 效果对比：混合方案准确率 87%，成本降低 60%

---

### Q4: 如果让你重新设计这个项目，你会做什么改动？

**回答要点**：
1. 监控体系：增加 Prometheus + Grafana，监控 API 延迟、错误率、缓存命中率
2. 向量化升级：用 BGE-M3 替换 Jaccard，提升语义召回准确率
3. 流式响应：支持 SSE 流式输出，降低用户感知延迟
4. A/B 测试框架：对不同 Prompt、模型配置进行 A/B 测试

**优先级**：监控 > 流式 > 向量化 > A/B 测试
